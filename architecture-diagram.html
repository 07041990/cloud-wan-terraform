Key Features

üîí
Network Segmentation
Production and Non-Production VPCs are completely isolated with no east-west traffic. Cloud WAN segment policies enforce isolation at the network layer.
üåê
Centralized Egress
All internet-bound traffic flows through a single egress VPC with centralized NAT Gateways and Network Firewall. No direct IGWs on spoke VPCs.
üõ°Ô∏è
GWLB Pattern
Gateway Load Balancer provides transparent, scalable firewall insertion. Traffic is automatically routed through Network Firewall for inspection.
üîç
Network Firewall
AWS Network Firewall with stateful inspection, domain filtering, IPS, and comprehensive logging. Blocks malicious domains and IPs.
ü§ù
Shared Services
Shared Services VPC is accessible from all segments for common services like DNS, monitoring, logging, and centralized management.
üè¢
Multi-Account
Supports multiple AWS accounts with cross-account resource sharing via AWS RAM. Centralized network management in hub account.
‚úÖ Success Criteria
  • Prod and Non-Prod Isolation: No east-west reachability between Production and Non-Production VPCs enforced by Cloud WAN segment policies
  • Shared Services Access: Shared Services VPC (10.3.0.0/16) is reachable from all intended segments only (Prod, Non-Prod)
  • Centralized Inspection: Inspection VPC receives traffic as per Cloud WAN segment policies for centralized monitoring
  • Centralized Egress: Spoke VPC instances have internet access only via centralized egress VPC (no direct IGWs)
  • Firewall Protection: Malicious or blocked domains/IPs are dropped and logged in Network Firewall
  • GWLB Deployment: Gateway Load Balancer pattern implemented for transparent firewall insertion
  • Route Validation: All spoke VPC route tables confirm central egress pattern (0.0.0.0/0 ‚Üí TGW)
